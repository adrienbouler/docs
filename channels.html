This feature is available only for accounts with channels feature enabled. Please contact us if you'd like to get access to it.

Channels feature allows you to create multiple branches of your application that can be targeted to different set of users. With this feature you can upload your application content once and then link it to many channels from the web panel without the need of transferring it again.

Pros of using channels instead of separate applications

  • You don’t need to upload the same data for each application separately, just do it once
  • Version linking is less error-prone than files uploading - if you feel confident about any version you have, you can link it to public channel easily
  • Easy version reverting - if you made a mistake, just create a new version and link it back to working game content
  • Shared data - channels and version are sharing the same settings, you don’t need to copy version labels or changelogs
  • Extended access control - each channel can have separate set of license keys

Example use case

You may have an application called My Game. It’s released publicly and you’re planning to release its new version with lots of changes, but you’re unsure if there won’t be any serious bugs inside of it. To solve that problem, you may want to release the new release only to limited audience.

With channels you can create a setup like this:

  • My Game
    • Public Release
    • Testing Release

The work flow may look like this:

  1. Create new version inside My Game group.
  2. Upload application content to My Game group.
  3. Create new version inside Testing Release channel.
  4. Link it to a version created inside My Game group.
  5. Publish it.
  6. Collect feedback.
  7. If everything looks good, create new version inside Public Release channel.
  8. Link the same version as before.
  9. Publish it to a wider audience.

Getting started

You can create a channel from your application’s page.

  1. Go to Applications page.
  2. Click on your application name. If you don’t have an application created yet, please visit Getting Started page to learn how to create one.
  3. At top-right corner click on the button.
  4. From the drop-down menu choose New Channel option.
  5. On the new form enter your new channel name.
  6. Click on the Create button.

You can repeat this process to create more channels if needed.

Creating first channel will convert your existing application into a group with built-in main channel, so in result you will end up with a channel that name you're choosing and a main channel.
A group is a entry on your applications list that includes multiple channels. It works like an application, but you cannot publish your versions directly from a group - you have to link a version to an existing channel and then publish it on that channel.

Channels list

If you now navigate to your application group you will see a brief list of channels on that page.

From there you can:

  • Navigate to channel’s page by clicking on the channel name or Edit button
  • Navigate to channel’s published version if available
  • Navigate to channel’s draft version if available
  • Delete the channel by clicking on Delete button

Uploading version content

You can upload version content by creating a new version from your’s application group page. This is the same one that is listing a brief channel list above.

  1. Navigate to Applications page.
  2. Click on your application name.
  3. Click on New Version button.
  4. Click on Import Content button.
  5. If asked, choose that you want to upload a ZIP file.
  6. On the upload form select a ZIP file with your game’s content.
  7. The upload and version processing will start automatically. Wait until it’s finished.
  8. Click on the Edit next to Label property and enter a label (e.g. 1.0)
  9. Click on the Persist button to persist this version.
Persisting a version means that it's ready to be linked with a channel. It's not available to anyone unless you do that.

Linking a version

Now when your version is persisted, you can now navigate to your channel and link it. Linking a version means assigning and eventually publishing it on a channel. It’s required if you’d like to make your content available.

  1. Click on the Overview tab or navigate to your application just like before.
  2. Click on your channel name. This channel will publish your newly created version.
  3. Click on New Version button.
  4. Click on Link to version button.
  5. From the Select Version drop-down choose a version that you’ve just created.
  6. Click on the Link button.

Now you will see a processing progress. It’s needed to be done since different channels are using separate encryption keys. When finished, you will see the information that version has been linked.

Now you can decide to either…

  1. Link to different version if you do not want to publish this link, or
  2. Publish this version to make it available to the players.

Giving the access to your users

When your channel’s version is published then you may want to make it available to certain users. To do that you have to navigate back to your channel’s page.

  1. Go to Applications page.
  2. Locate your application on the list.
  3. Click on the channel name you’d like to share.
  4. On the channel’s page click on the Get Launcher button.

The launcher that you now have allows to access that specific channel. This also means that if you’d like anyone to use different channel, you need to share with them a new launcher.

You just need to share the launcher once to those who don't have it. Launcher will automatically upgrade locally installed applications when new version is available on a channel.
Please be aware that having the access to two or more channels requires to install the application in separate locations on your disk drive. Do not try to overwrite existing launcher with a new one.
