Tools

PatchKit Tools is set of programs that can be used to access PatchKit functionality through command line.

Installation

PatchKit Tools require Ruby installation (also it must be added to the PATH variable so ruby command is available).

To install PatchKit Tools clone repository from GitHub and add it's location to the PATH variable.

Usage

Every tool is started by patchkit-tools command. The working directory is set to location from which script was called.

Syntax

patchkit-tools [tool_name] [tool_arguments]

Displaying help and tools list

patchkit-tools --help

Displaying specific tool help

patchkit-tools [tool_name] --help

Updating application with version diff

You can use PatchKit Tools to update your application by creating a version diff. The advantage of this method is that you're not uploading whole version content - it works like a patch but in a opposite way.

Note that this method works only if your application contains at least one published version.

Guide

  1. Create a new draft version

    Command

    patchkit-tools create-version --secret [app_secret] --apikey [api_key] --label [new_version_label]
    • [app_secret] - your application secret that can be found on application page
    • [api_key] - your account api key that can be found on "My Profile" page

    Result

    Sending request...
Result: {"id"=>[new_version_id], "label"=>"[new_version_label]", "changelog"=>nil, "publish_time"=>-1, "content_guid"=>nil, "diff_guid"=>nil, "draft"=>true}
Done!
    • [new_version_id] - id of newly created version

  2. Download previous version signatures

    Command

    patchkit-tools download-version-signatures --secret [app_secret] --apikey [api_key] --version [new_version_id - 1] --output [signatures_file]
    • [new_version_id - 1] - it's simply [new_version_id] decreased by one
    • [signatures_file] - path to file where signatures will be saved

    Result

    |===========================================================================|
Done!

  3. Create diff

    Command

    patchkit-tools diff-version --files [new_version_directory] --signatures [signatures_file] --diff [diff_file] --diffsummary [diff_summary_file]
    • [new_version_directory] - path to new version files
    • [diff_file] - path to file where diff will be saved
    • [diff_summary_file] - path to file where diff summary will be saved

    Result

    Unpacking signatures...

Done!

  4. Upload diff

    Command

    patchkit-tools upload-version --secret [app_secret] --apikey [api_key] --mode diff --version [new_version_id] --file [diff_file] --diffsummary [diff_summary_file]

    Result

Done!

  5. Publishing version

    Command

    patchkit-tools publish-version --secret [app_secret] --apikey [api_key] --version [new_version_id]
